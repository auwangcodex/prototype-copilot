From idea to research-backed prototype in one session

Prototype Co-Pilot automates the full PM workflow — competitive research, behavioral design, interactive prototyping, and stakeholder packaging — as Claude Code slash commands.

Get Started See the Demo
"I'm pivoting to PM. Every startup interview asks for a prototype or case study that demonstrates product thinking. But building one from scratch takes days — research, design, code, the pitch. Most people skip the research and just vibe-code something that looks good but has no depth."

Prototype Co-Pilot was born from this exact problem. It started as a workflow for a real Whoop PM interview (AI fitness coach), then was generalized into reusable skills, tested on BoldVoice and Canvas LMS, and refined through comparison with tools like Lovable. The result: a pipeline that produces prototypes grounded in real user research, not plausible-looking guesses.

Pipeline

Four skills. One coherent workflow.

Each skill chains into the next, building on the outputs of the previous phase. You make the strategic decisions at human-in-the-loop checkpoints.

Step 1

/pm-research

Scrapes 50+ app reviews, analyzes 3-5 competitors, runs live product walkthroughs via Claude in Chrome, applies Branching 5 Whys, and audits underutilized data.

→ research-brief.md
→ design-tokens.md
Step 2

/pm-design

Presents 2-3 behavioral frameworks grounded in research. You choose. Then generates persona specs with coach personalities, day-in-life scenarios, and sample dialogue.

→ design-spec.md
→ persona-spec.md
Step 3

/pm-build

Builds a zero-build-step HTML prototype (React 18 from CDN). Single file, opens in any browser. Interactive components with real design tokens from the research phase.

→ prototype.html
→ (open in browser)
Step 4

/pm-package

Generates stakeholder email (500-800w), blog case study (1500-2500w), and a Lovable.dev build spec with verified data for production handoff.

→ email.md
→ blog.md
→ lovable-prompt.md
Why This Matters

The product thinking layer that vibe-coding skips

Most AI prototyping tools go from prompt to code. Prototype Co-Pilot adds the research and design rigor that makes prototypes defensible in an interview or stakeholder review.

🔎

Live Product Walkthroughs

Claude navigates the actual product in Chrome, documenting every screen, extracting real UI copy, and capturing the exact user journey. Your prototype matches the real thing because it was built from the real thing.

🧠

Behavioral Frameworks

Every prototype is grounded in a behavioral framework (Hook Model, Cognitive Load Reduction, Self-Determination Theory) chosen based on the research findings, not a random template.

👥

Human-in-the-Loop

The agent pauses at critical decisions: framework selection, persona choice, layout variant. You make the strategic calls. The AI does the heavy lifting. Better outcomes than full automation.

🚀

Research-Grounded Lovable Export

The pm-package skill generates a Lovable.dev build spec packed with verified data from your research. Lovable builds with facts, not hallucinated placeholder content.

Get Started

Install in 60 seconds

Three commands. Then open Claude Code and start with /pm-research.

Terminal 
# Clone the repo
git clone https://github.com/auwangcodex/prototype-copilot.git ~/prototype-copilot

# Run the installer (symlinks skills to ~/.claude/skills/)
cd ~/prototype-copilot && ./install.sh

# Start a new Claude Code session
claude

# Run the full pipeline on any product
/pm-research Canvas --idea "AI chatbot for course navigation"
/pm-design Canvas
/pm-build austin --project Canvas
/pm-package --format all --project Canvas
