"I'm pivoting to PM. Every startup interview asks for a prototype that demonstrates product thinking. But building one from scratch takes days — research, design, code, the pitch. Most people skip the research and just vibe-code something that looks good but has no depth."

Prototype Co-Pilot was born from this exact problem. It started as a workflow for a real Whoop PM interview, then was generalized into reusable skills that orchestrate third-party MCPs (Firecrawl for web scraping, Mobbin for UI references) and community skills. Tested on BoldVoice and Canvas LMS, refined through comparison with Lovable. The result: a pipeline that produces prototypes grounded in real user research, not plausible-looking guesses.

Pipeline

Four skills. One coherent workflow.

Each skill chains into the next. You make the strategic decisions at human-in-the-loop checkpoints.

Step 1

/pm-research

Orchestrates Firecrawl MCP to scrape 50+ app reviews, analyzes 3-5 competitors, runs live product walkthroughs via Claude in Chrome, pulls UI refs from Mobbin MCP, applies Branching 5 Whys, and audits underutilized data.

→ research-brief.md
→ design-tokens.md
Step 2

/pm-design

Presents 2-3 behavioral frameworks grounded in research. You choose. Then generates user personas with goals, scenarios, and sample interactions tailored to the product.

→ design-spec.md
→ persona-spec.md
Step 3

/pm-build

Builds a zero-build-step HTML prototype (React 18 from CDN). Single file, opens in any browser. Interactive components with real design tokens from the research phase.

→ prototype.html
→ (open in browser)
Step 4

/pm-package

Generates stakeholder email (500-800w), blog case study (1500-2500w), and a Lovable.dev build spec with verified data for production handoff.

→ email.md
→ blog.md
→ lovable-prompt.md
Features

The product thinking layer
that vibe-coding skips.

Most AI prototyping tools go from prompt to code. Prototype Co-Pilot adds research, design rigor, and a composable ecosystem of third-party MCPs and skills that make prototypes defensible in an interview.

🔌

Third-Party MCP Ecosystem

Integrates Firecrawl for JS-rendered page scraping, Mobbin for UI screenshot references, and any MCP server you configure. The pipeline leverages external tools Claude Code can't access alone — turning it from a code generator into a research-powered system.

🧠

Composable Skill Architecture

Each skill (research, design, build, package) is a standalone SKILL.md file. Add community skills, swap in your own frameworks, or chain third-party skills from other authors. The pipeline is open and extensible by design.

⚖️

Live Product Walkthroughs

Claude navigates the actual product in Chrome, documenting every screen, extracting real UI copy, and capturing the exact user journey. Your prototype matches the real thing because it was built from the real thing.

🤝

Human-in-the-Loop

The agent pauses at critical decisions: framework selection, persona choice, layout variant. You make the strategic calls. The AI does the heavy lifting.

💎

Behavioral Frameworks

Every prototype is grounded in a behavioral framework (Hook Model, Cognitive Load Reduction, Self-Determination Theory) chosen based on research findings, not a random template.

🚀

Lovable Export

The pm-package skill generates a Lovable.dev build spec packed with verified data from your research. Lovable builds with facts, not hallucinated placeholder content.

Prototype Co-Pilot handles research & design thinking. Lovable handles production engineering. They're complementary — the pm-package skill bridges them.

The key differentiator: Prototype Co-Pilot doesn't just prompt Claude — it orchestrates an ecosystem of third-party MCP servers (Firecrawl, Mobbin, custom scrapers) and composable skills that any developer can extend. Raw Claude Code has access to none of this context without manual configuration. The skills encode the workflow intelligence that turns a general-purpose coding agent into a domain-specific research machine.

Get Started

Install in 60 seconds.

Three commands. Then open Claude Code and start with /pm-research.

Terminal 
# Clone the repo
git clone https://github.com/auwangcodex/prototype-copilot.git ~/prototype-copilot

# Run the installer (symlinks skills to ~/.claude/skills/)
cd ~/prototype-copilot && ./install.sh

# Start a new Claude Code session
claude

# Run the full pipeline on any product
/pm-research Notion --idea "AI writing assistant"
/pm-design Notion
/pm-build <persona-name> --project Notion    # use persona name from /pm-design
/pm-package --format all --project Notion
